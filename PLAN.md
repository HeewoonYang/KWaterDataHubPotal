# 외부 사용자 사용신청/등록 온라인 폼 및 승인 프로세스

## 개요
로그인 화면에서 접근 가능한 **외부 사용자 사용신청 폼**과, 관리자가 시스템관리 내에서 확인하는 **승인 프로세스 화면**을 추가합니다.

---

## 1. 외부 사용자 사용신청 폼 (로그인 화면에서 접근)

### 위치
- 로그인 화면(`#loginOverlay`) 하단에 **"외부 사용자 신청"** 링크 추가
- 클릭 시 로그인 카드가 신청 폼 카드로 전환 (SPA 방식)

### 폼 구성 (신청서 입력 필드)
| 구분 | 필드 | 유형 | 비고 |
|------|------|------|------|
| 기본정보 | 성명 | text | 필수 |
| | 소속기관 | text | 필수 |
| | 부서/팀 | text | 필수 |
| | 직위/직급 | text | |
| 연락처 | 이메일 | email | 필수, 본인인증용 |
| | 휴대폰번호 | tel | 필수 |
| 신청정보 | 사용목적 | select | 데이터 조회/API 연동/공동 연구/기타 |
| | 상세 사용목적 | textarea | 필수 |
| | 희망 접근 데이터 | checkbox 목록 | 수문/수질/댐/수도/기상 등 |
| | 사용 예정기간 | 시작~종료 date | |
| 첨부 | 사업자등록증/재직증명 | file upload UI(데모) | |
| 동의 | 개인정보 수집·이용 동의 | checkbox | 필수 |
| | 데이터 활용 약관 동의 | checkbox | 필수 |

### 제출 후 동작
- "신청이 접수되었습니다" 토스트 + 접수번호 표시
- 로그인 화면으로 자동 복귀

---

## 2. 관리자 승인 화면 (시스템관리 > 외부사용자 신청관리)

### 위치
- 사이드바 시스템관리 아코디언(`#acc-system`)에 **"📋 외부사용자 신청관리"** 메뉴 추가
- screen id: `sys-ext-register`
- `pages/system.html`에 새 screen div 추가

### 화면 구성

#### Row 1: KPI 카드 4개
- 전체 신청 (누적)
- 승인대기 (주황)
- 승인완료 (초록)
- 반려 (빨강)

#### Row 2: AG Grid 신청 목록
컬럼: 접수번호 | 신청일 | 성명 | 소속기관 | 사용목적 | 상태(badge) | 처리일 | 관리(상세 버튼)

#### Row 3: 신청 상세 화면 (sys-ext-register-detail)
- 상세 버튼 클릭 → `sys-ext-register-detail` 화면 이동
- 신청자 정보 전체 표시 (2컬럼 레이아웃)
- 처리 이력 타임라인
- **승인/반려** 버튼 + 반려 사유 입력
- 승인 시: 계정 자동 생성(데모) + 임시 비밀번호 발급 표시
- 반려 시: 반려 사유 모달

## 3. 수정 파일 목록

| 파일 | 변경 내용 |
|------|-----------|
| `index.html` | 로그인 화면에 "외부 사용자 신청" 링크 + 신청 폼 HTML, 사이드바에 메뉴 추가 |
| `pages/system.html` | `sys-ext-register` 화면 + `sys-ext-register-detail` 상세 화면 |
| `js/main.js` | 신청 폼 전환/제출 함수, AG Grid 초기화, 승인/반려 처리 함수, RBAC 반영 |
| `css/style.css` | 신청 폼 스타일 (기존 패턴 재활용, 최소 추가) |

- RBAC_MATRIX screens 배열 + SCREEN_PARENT_MAP에 추가
