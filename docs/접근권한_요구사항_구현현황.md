## Sub_No 001 — 접근권한 조회

> 사용자별/그룹별 접근권한 정보를 체계적으로 관리 및 조회할 수 있는 권한 관리 시스템 구현

### 구현 여부: ✅ 구현됨

### 해당 화면

| 화면 ID | 화면명 | 메뉴 경로 |
|---------|--------|----------|
| `screen-sys-user` | 사용자 관리 | 시스템관리 → 사용자 관리 |
| `screen-sys-role` | 권한 및 역할관리 | 시스템관리 → 권한 및 역할관리 |

### 구현 내용

#### 1) 사용자 관리 화면 (`screen-sys-user`)
- **AG Grid** (`ag-grid-sys-user`): 12개 컬럼
  - 이름, 사번, 부서, 직급, **역할**, **접근등급**, 상태, SSO, 최종로그인, 가입일, 관리(액션)
- **역할 배지**: 5종 (시스템관리자, 데이터관리자, 데이터엔지니어, 일반사용자, 외부사용자)
- **접근등급 배지**: 5종 (전체, 할당됨, 등급별, 3등급만, 대기)
- **필터**: 역할별/상태별 필터 제공
- 샘플 데이터 12명 포함

#### 2) 권한 및 역할관리 화면 (`screen-sys-role`)
- **RBAC 권한 매트릭스 테이블**: 9개 역할 × 기능영역 매핑 조회
- 역할별 접근 가능 화면 목록 및 권한 레벨 일괄 조회

#### 관련 JS

| 함수명 | 역할 |
|--------|------|
| `initSystemGrids()` | 사용자 관리 AG Grid 초기화 |
| `RBAC_MATRIX` | 9개 역할별 접근 가능 화면 정의 |
| `RBAC_SCREEN_PERMS` | 역할별 화면별 권한 레벨 정의 |

## Sub_No 002 — 접근권한에 따른 사용자 조회

> 권한별 사용자 목록 조회 및 권한 변경 내역 이력 관리 기능 제공

### 구현 여부: ✅ 구현됨

### 해당 화면

| 화면 ID | 화면명 | 메뉴 경로 |
|---------|--------|----------|
| `screen-sys-user` | 사용자 관리 | 시스템관리 → 사용자 관리 |
| `screen-sys-role` | 권한 및 역할관리 | 시스템관리 → 권한 및 역할관리 |

### 구현 내용

#### 1) 권한별 사용자 필터링
- AG Grid 내 **역할 컬럼 필터**: 역할별 사용자 목록 필터링 가능
- **접근등급 컬럼 필터**: 등급별 사용자 조회 가능
- **상태 컬럼 필터**: 활성/비활성/승인대기/잠김 상태별 조회

#### 2) 권한 변경 이력 관리
- **저장소**: `localStorage` → `PERM_HISTORY` 키
- **기록 항목**: 날짜, 역할, 화면명, 이전권한 → 새권한
- **보관**: 최근 20건 유지
- **모달**: `roleAuditModal` (권한 변경 감사로그 조회)

#### 관련 JS

| 함수명 | 역할 |
|--------|------|
| `addPermHistory(roleKey, screenId, oldPerm, newPerm)` | 권한 변경 이력 기록 |
| `updatePermHistory()` | 변경 이력 UI 갱신 |
| `openRoleModal('roleAuditModal')` | 감사로그 모달 열기 |

## Sub_No 003 — 접근권한 사용자 추가/삭제

> 역할/그룹 기반으로 사용자 접근권한의 부여 및 회수, 자동 로그 기록

### 구현 여부: ⚠️ 부분구현

### 해당 화면

| 화면 ID | 화면명 | 메뉴 경로 |
|---------|--------|----------|
| `screen-sys-user-detail` | 사용자 상세 | 사용자 관리 → 상세 |
| `screen-sys-role` | 권한 및 역할관리 | 시스템관리 → 권한 및 역할관리 |

### 구현 내용

#### 1) 사용자 상세화면 (`screen-sys-user-detail`)
- **기본정보 표시**: 이름, 사번, 부서, 직급, 이메일, 휴대폰
- **역할 테이블**: 복수 역할 할당 현황 및 권한 레벨 표시
- **액션 버튼** (권한 수준별 표시):
  - `data-perm="write"` → "수정" 버튼
  - `data-perm="admin"` → "역할 변경" 버튼
  - `data-perm="admin"` → "계정 비활성화" 버튼 (빨강)
- **역할 해제 버튼**: 개별 역할 제거 (`data-perm="admin"`)

#### 2) 자동 로그 기록
- 권한 변경 시 `PERM_HISTORY`에 자동 기록
- ERP 동기화 시 감사로그 자동 기록

#### 관련 JS

| 함수명 | 역할 |
|--------|------|
| `openUserDetail(data)` | 사용자 상세 화면 로드 |
| `addPermHistory()` | 변경 이력 자동 기록 |

### 미구현 사항
- ❌ 사용자 신규 추가 폼 (UI만 존재, API 미연동)
- ❌ 역할 부여/회수 실제 처리 (버튼만 존재, 로직 미완성)
- ❌ 그룹 단위 일괄 권한 부여/회수

## Sub_No 005 — 접근권한과 데이터 매치 설정/수정 기능

> 권한-데이터 매핑 인터페이스 제공, 수정 이력 관리 및 승인 프로세스 지원

### 구현 여부: ✅ 구현됨

### 해당 화면

| 화면 ID | 화면명 | 메뉴 경로 |
|---------|--------|----------|
| `screen-sys-perm` | 화면별 권한설정 | 시스템관리 → 화면별 권한설정 |

### 구현 내용

#### 1) 권한-데이터 매핑 인터페이스
- **역할 선택 드롭다운**: RBAC_MATRIX 9개 역할
- **AG Grid** (`ag-grid-sys-perm`): 8개 컬럼
  - 메뉴그룹, 화면명, Screen ID, CRUD 권한 표시(✓/—), 현재권한, 권한설정(select)
- **권한 레벨**:
  - `read` (조회) / `write` (조회+등록) / `manage` (조회+등록+수정+삭제) / `admin` (전체관리)
- 커스텀 설정 시 행 배경색 노란색(`#fff8e1`) 변경으로 시각적 구분

- **저장소**: `localStorage` → `CUSTOM_SCREEN_PERMS`, `PERM_HISTORY`
- **이력 형식**: `{ date, role, screen, screenName, from, to }`
- **UI 표시**: 최근 5건 변경 이력 화면 하단에 표시

#### 3) KPI
- 전체 역할 수, 화면 수, 커스텀 설정 수

#### 관련 JS

| 함수명 | 역할 |

| `permSelectRenderer(params)` | 권한 설정 select 렌더러 |
| `onPermSelectChange(el, screenId)` | 권한 변경 시 즉시 저장 |
| `updatePermKPIs(roleKey)` | KPI 수치 계산 |
| `savePermChanges()` | 권한 설정 확인 |
| `resetPermChanges()` | 커스텀 권한 초기화 |
| `getCustomScreenPerms()` | localStorage에서 커스텀 권한 조회 |
| `saveCustomScreenPerms(data)` | localStorage에 커스텀 권한 저장 |
| `getScreenPerm()` | custom > 기본매트릭스 우선순위로 권한 조회 |

### 미구현 사항

## RBAC 시스템 구조

### 역할 체계 (9개)

| 그룹 | 역할명 | 키 | 기본 권한 |
|------|--------|-----|----------|
| 수공직원 | 일반사용자 | `수공직원\|일반사용자` | read |
| 수공직원 | 부서관리자 | `수공직원\|부서관리자` | write |
| 수공직원 | 데이터스튜어드 | `수공직원\|데이터스튜어드` | manage |
| 협력직원 | 조회사용자 | `협력직원\|조회사용자` | read |
| 협력직원 | 개발자 | `협력직원\|개발자` | write |
| 데이터엔지니어 | 파이프라인 | `데이터엔지니어\|파이프라인` | manage |
| 데이터엔지니어 | 운영 | `데이터엔지니어\|운영` | manage |
| 데이터엔지니어 | DBA | `데이터엔지니어\|DBA` | manage |
| 관리자 | 시스템관리자 | `관리자\|시스템관리자` | manage/admin |

### 권한 레벨

```
read(1)   — 조회만 가능
write(2)  — 조회 + 등록
manage(3) — 조회 + 등록 + 수정 + 삭제
admin(4)  — 전체 관리 (역할 변경, 계정 비활성화 등)
```

### 권한 제어 방식
- HTML 요소에 `data-perm="write|manage|admin"` 속성 부여
- `navigate()` 함수에서 화면 전환 시 권한 확인 후 `.perm-hidden` 클래스 토글
- `getScreenPerm()`: 커스텀 설정 → 기본 매트릭스 순서로 권한 조회
